\section{はじめに}
\label{sec:introduction}

大規模言語モデル (LLM) の応答を実運用で利用する場面が急速に拡大する
なか、その出力品質を自動的かつ再現可能に評価する枠組みの重要性が
増している。翻訳や要約など参照応答を前提とするタスクでは、BLEU や
ROUGE に代表される表層類似度指標や、BERTScore・BLEURT などの埋め込み
類似度指標が広く用いられてきた。さらに近年は、参照応答を必要とせず
柔軟な基準で評価可能な G-Eval や LLM-as-Judge といった LLM 駆動の
評価手法が急速に普及している。

しかしこれらの既存指標はいずれも、応答を単一スカラに収束させる設計
であるか、あるいは評価軸の分離を prompt 設計に委ねる構造を取るため、
応答の「どの側面が壊れたか」を計算機構の段階で分離することが難しい。
特に、応答が構造として成立しているか (構造完全性) と、問いの核心
命題を拾えているか (命題カバレッジ) という 2 種類の失敗モードは
本質的に独立した軸であるが、既存指標では両者が単一スコアに混合されて
しまい、判定の根拠を読み解くことが困難である。加えて、judge LLM に
依存する手法はモデル更新に伴う評価変動という再現性の問題も抱える。

本稿では、LLM 応答の意味的整合性を構造完全性 (S 軸) と命題カバレッジ
(C 軸) の 2 軸で分離評価する PoR/$\Delta E$ フレームワークを提案し、
HA48 における予備検証を通じて人手評価との対応を確認する。主要な
貢献は以下の 3 点である:

\begin{enumerate}
  \item 応答の位置を 2 次元座標 $\mathrm{PoR} = (S, C)$ として
        architectural に分離表現し、理想応答からの距離指標
        $\Delta E$ を定式化する。
  \item 意味損失関数 $L_{\mathrm{sem}}$ を 3 項 (命題損失 $L_P$、
        用語捏造損失 $L_F$、因果構造損失 $L_G$) に分解することで、
        判定が具体的にどの誤差項に駆動されたかを解釈可能な
        診断情報として提供する。
  \item 48 件の人手アノテーションデータ (HA48) を用いた予備検証に
        より、提案指標が人手評価との間で統計的に有意な相関
        (Spearman $\rho = -0.482$, $p < 0.001$) を示すことを実証する。
\end{enumerate}
