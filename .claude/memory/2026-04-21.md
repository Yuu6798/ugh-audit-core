## 技術メモ (次回参照用)

### docs 直接 parse test pattern

`tests/test_validation_ci.py` で採用した「docs markdown table を regex parse
して test 値として使う」pattern は、docs と test の乖離を構造的に防ぐ強力な
idiom。同パターンを他の指標表 (thresholds.md など) に展開する余地あり。

### reconstruct_hit_sources の段階フォールバック設計

Evidence のバージョン差分を吸収する `reconstruct_hit_sources(evidence)` は
「優先順位付き getattr チェーン」が鍵。api 層で `if hasattr(...) else {}` を
書きたくなった時は常にこのパターンで legacy 耐性を持たせる方が安全。

### HA48 exact cardinality lock

`EXPECTED_HA48_N = 48` 定数 + `assert len(ids) == EXPECTED_HA48_N` の組み合わせで
accidental truncation を完全検知。dataset 拡張時は定数と CSV を同一コミットで
更新する運用。他の baseline dataset (HA20, HA63) にも同パターン適用可能。
