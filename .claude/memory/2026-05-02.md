## 修正・訂正

- **Repo を取り違え**: PR1 (docs only) を最初 svp-video-pipeline に push してしまった。
  brief が `svp_rpe` namespace 表記だったが実際は ugh-prompt-engine。user 訂正で
  ugh-prompt-engine 側で やり直し、svp-video-pipeline 側の wrong branch は
  user に手動削除を依頼。
- **`File2Beats` vs `Audio2Beats`**: beat_this >= 1.1 の実 API では in-memory
  signal には `Audio2Beats(checkpoint_path=..., dbn=False)(signal, sr)` を使う。
  最初 `File2Beats` を使っていたが fake backend では通る (実環境で TypeError)。
  user 指摘で adapter + fake 両方を `Audio2Beats` ベースに修正、test fake は
  `File2Beats` 不在で adapter が逆戻りしようとすると AttributeError で即落ちる
  構造にした。
- **`panns_inference.labels` のサブモジュール扱い**: 実 wheel では root attribute
  (re-export from `panns_inference.config`) で submodule じゃない。
  `importlib.import_module("panns_inference.labels")` は実環境で必ず ImportError
  になり adapter が動かない (fake backend が偽 submodule を作っていたため CI で
  見逃し)。Codex P1 reviewer 指摘で `getattr(panns_root, "labels")` →
  `panns_inference.config.labels` フォールバックに修正、fake も attribute layout
  に揃えて real install と CI の動作を一致させた。
- **panns に sample_rate を metadata だけ記録、resample なし**: Cnn14 は
  sample_rate=32000 baked-in で内部 resample しない。44.1/48 kHz の audio を
  渡すと out-of-distribution signal を silently 推論していた。Codex P2 指摘で
  `librosa.resample` で 32 kHz に揃える + `inference_config.target_sample_rate`
  に provenance を記録。32 kHz 入力時は no-op fast path 維持。
- **PR2 で `RPEBundle.model_dump()` に `learned_annotations: null` 常時混入**:
  既存 `examples/expected_output/*/rpe.json` snapshot drift を引き起こす。
  `PhysicalRPE.omit_empty_stem_rpe` パターンの `@model_serializer(mode="wrap")`
  を `RPEBundle` に追加して None 時に key を pop。self-review で先に気づき
  user 指摘前に修正済だった (user は「対応した?」確認に至った)。
- **PR4 で `top_k=True` (bool) が validation 素通り**: `bool` は `int` の
  サブクラスで `True <= 0` は False。silently top-1 として動作していた。
  self-review で発見、`isinstance(top_k, bool)` を最初に弾く `_validate_top_k`
  ヘルパーに切り出し。
- **`LearnedAudioLabel.evidence` が `SemanticLabel.evidence` と命名衝突**:
  policy が「rule-based evidence と learned 推定を混同するな」と言っているの
  に同じ field 名を使っていた。self-review で `notes` に rename、docstring に
  「`evidence` という名前は SemanticLabel の epistemic weight 専用、learned
  は provenance hint なので `notes` が正しい」と明記。
- **PR5 `_extract_note_events_field` で `result[2]` の sequence 検証なし**:
  fake が `({}, None, "garbage")` を返すと文字列が iterate されて per-character
  "unexpected shape" error になる。self-review で `isinstance(note_events,
  (list, tuple))` の中間ガードを追加して contract-level で弾くようにした。

## 成功パターン

- **Adapter spike テンプレ**: `LearnedModelUnavailable` (extra not installed) と
  `LearnedModelIncompatible` (installed but API shifted) の二段階 error class、
  `_load_X()` lazy import、provenance metadata に `(name, version, provider,
  task, license, weights_license)` を埋める、fake backend を sys.modules
  monkeypatch で差し込み real install ゼロで CI green。3 adapter
  (beat_this / panns / basic_pitch) で完全に再利用、4 番目以降も同パターンで
  追加できる。
- **Sentinel test for leak detection**: sentinel 文字列を adapter 出力の
  source_model / label / time_sec などに注入し、`generate_svp().model_dump_json()`
  に含まれないことを assert する。policy 違反 (learned 出力が SVP に漏れる) の
  自動検知ライン。3 adapter すべてで同手法で pin。
- **Self-review → 即修正の default ルール化**: user が「以後セルフレビューで
  見つけたものは修正対応してください」を宣言、PR 直後に diff を読んで findings
  を 1〜複数 commit で潰すワークフローを採用。PR4 で 5 件、PR5 で 1 件を即解決。
  PR description で残置事項を明記して deferral の意図も透明化。
- **License hedging の一貫パターン**: `LearnedModelInfo.license` は code license
  のみ宣言 (例: "MIT" / "Apache-2.0")、`weights_license = None` で artifact
  license を断定しない、`license_metadata[name]` に "code license; model
  artifact provenance requires upstream verification" の but-text を入れる。
  test では `assert info.weights_license is None` と `"weights" in
  license_text.lower()` の両方を assert して後退禁止を pin。3 adapter で完全一致。
- **段階分割 PR**: policy doc → schema scaffold → 3 adapter の独立 5 PR。
  PR2 で omit-None serializer を入れたことで PR3〜5 が安全に schema を additive
  拡張できた。各 PR 単体で revert 可能、blast radius 最小。
- **`LearnedModelUnavailable` の package root 一元化**: PR4 で `learned/__init__.py`
  に集約、各 adapter で `from svp_rpe.rpe.learned import LearnedModelUnavailable`
  + `__all__` で再 export。caller が `except LearnedModelUnavailable` 1 つで
  全 adapter からの error を catch できる。`TestErrorClassUnification::*` で
  `assert FromBeat is FromPanns is FromBasicPitch is Unified` を pin。

## 設計判断

- **`schema_version` を learned 追加 field で bump しない**: 全 field が
  `Optional[...]` または `Field(default_factory=list)` で backward compatible。
  PR2/PR3/PR5 で `time_events` / `note_events` を additive に積んだが
  `LearnedAudioAnnotations.schema_version` は "1.0" 維持。既存 PR2 era JSON が
  re-parse 可能であることを test で pin。
- **resample は adapter 側で吸収**: panns の Cnn14 は 32 kHz 必須なので caller
  に押し付けず adapter が `librosa.resample` で吸収。`inference_config` に
  `sample_rate` (input) と `target_sample_rate` (model に渡した値) の両方を
  provenance 記録、auditable に。
- **basic-pitch は path 入力のみ**: upstream が `predict(audio_path)` 中心の
  API。in-memory variant は後続 PR で必要になったら追加、spike では path
  入力で十分。
- **`File2Beats` ではなく `Audio2Beats`**: 既に decoded signal を持っているのに
  disk re-decode を強制する `File2Beats` を使う理由がない。adapter docstring
  でこの選択理由を明記、test fake は `Audio2Beats` のみ公開して `File2Beats`
  への逆戻りを CI で検出。
- **`_extract_note_events_field` の type 検証は 3 段階で逐次**: outer
  container (tuple/list) → length (>= 3) → inner field (list/tuple) を順番に
  弾くことで、エラーメッセージが contract のどの level で違反したかを正確に
  指す。Pydantic の `ValidationError` だと中間情報が missing なので手書き
  guard が読みやすい。
- **per-call model instantiate を直さない**: `AudioTagging(...)` は ~80MB
  weights を毎回ロードするが、cache を入れると test isolation を壊すリスクが
  高い (sys.modules を monkeypatch する fake backend と相性が悪い)。pipeline
  統合 PR で `audio_tagging: Optional[AudioTagging] = None` の caller-managed
  lifecycle に切り替える方が筋が良いと判断、現 PR では module docstring に
  TODO として明記。
