## 設計判断 (M4 Video 層、全 10 項目確定)

1. **480p 価格**: 推定値ハードコード禁止。`cost_note: "pending_official_rate"`
   で `cost_usd=None` 併記、実 API 実行後に本値で差し替え
2. **`no_video=False` 既定化**: 採用。`Pipeline.run()` docstring 冒頭に
   "WARNING: ~$1.5 per call at standard 720p" を明記。M3 README の手動実行
   スニペットから `no_video=True` を削除
3. **リトライ semantics**: `MAX_RETRIES=2` は "initial + 2 retries = 最大 3 試行"
   の意味と明記。`test_max_retries_exceeded` は 3 回連続失敗で raise
4. **プロンプト言語**: M3 `render_image_prompt` の方針踏襲。セクション見出し
   英語、`por_core`/`grv_anchor`/`subject_motion` 要素内容は原文 (日本語) ママ
5. **duration バリデーション**: `VideoGenerator.generate()` 入口で
   `duration_seconds not in range(4, 16)` を fail-fast (コスト発生前に止める)
6. **画像アップロード TTL**: `_upload_image` docstring に「同一セッション内の
   利用前提」と記載
7. **観察レポート保存先**: M3 パターン踏襲、`svp_pipeline/README.md` に
   "M4 Observation Snapshot" セクション追記
8. **`[Image1]` コメント**: `render_motion_prompt` 内に
   `# [Image1] is Seedance r2v's official reference notation for image_urls[0]` を
   記載
9. **motion_layer エッジケース**: `test_render_motion_prompt_minimal_motion_layer`
   を追加 (`subject_motion=[]`, `temporal_anchors=[]` 等でも crash しない)
10. **branch 命名**: `claude/*` (Claude) / `codex/*` (Codex) の実装主体別 prefix
    使い分けを svp の CLAUDE.md に明記
