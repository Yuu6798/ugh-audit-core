## 設計判断
- **VPS vs クラウド DB**: VPS (Railway) を選択。理由: API サーバー + DB が一体で動き、コード変更ゼロ。クラウド DB だけ移行しても API サーバーの置き場所が別途必要
- **PyTorch CPU 版**: GPU 不使用のため CPU 版で機能同一、イメージサイズ 5.5GB → ~3.5GB に削減
- **プロキシモード**: ローカル MCP → Railway API への転送方式を採用。ローカル DB を廃止し Railway DB に一本化。Claude は認証なし MCP に接続不可のため、ローカル MCP をプロキシとして残す必要がある
- **limit クランプ**: history=500, drift=1000 を上限に設定。負の値で全テーブルダンプされる脆弱性を防止
