### PR #109 merged (Steiger's Z pairwise significance)

user 質問「UGH は既存ベンチと比較して有意性を確認できたと言えるか?」への
正直回答として Steiger's Z (1980) dependent correlations 検定を実装、
pairwise 比較を pairwise table + 論文主張境界として明文化。

結果 (Steiger's Z p 値):

| dataset | vs BLEU | vs BERTScore | vs SBert |
|---|---|---|---|
| HA20 | 0.133 | 0.234 | **0.079** (10% 水準で有意方向) |
| HA48 | 0.325 | 0.363 | 0.168 |

**全ペアで α=0.05 非有意**。n=20/48 の検出力不足により、Δρ=+0.15〜+0.34
の medium 以上 effect size でも有意に届かない。ただし HA20 UGHer vs
SBert の p=0.079 は「UGHer の強さは内部 SBert 単独ではない」命題を
弱く統計的に支持する。

## 技術メモ (次回参照用)

### Steiger's Z 実装パターン

同一サンプル内で複数指標が共通の target (O スコア) とどれだけ相関するかの
差の検定。`analysis/baseline_comparison.py` の `steiger_z_test()` +
`pairwise_steiger_table()` を参照。Fisher z 変換 + 共変量 `r_xz` で分散
共分散調整する標準形。scipy に無いので手実装。Psychological Bulletin
87(2) 245-251 に準拠。

### 「honest claim boundary の明文化」という deliverable

論文査読耐性は「検定結果そのもの」と同じくらい「書ける主張 / 書けない
主張の境界を文書で固定する」ことで担保される。今日 Codex が flag した
overclaim (「個別 ρ の有意性は全指標で確保」) は、境界を曖昧にしたまま
結論を急いだ典型例。**indicator 別に p 値を具体列挙する** ことで
overclaim を構造的に防げる。
