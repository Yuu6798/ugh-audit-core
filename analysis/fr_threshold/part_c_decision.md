## 4. 採用判断

### 推奨: **fr = 0.30**

#### 根拠

1. **回収効果**: +9件 (正当6件, 偽ヒット3件)。到達率 58.4% → 61.3% (+2.9pp)
2. **偽ヒット3件**: 判定基準「1-3件でdecision影響なし → 採用可」に該当
   - q016:p2 — decision変化なし
   - q030:p0 — regenerate→rewrite (1段改善、偽ヒットだが回答品質は低くない)
   - q065:p1 — decision変化なし
3. **ρ維持**: 19/20 (0.95)。HA20の方向性一致に劣化なし
4. **f4維持**: 45/102。検出層フラグに変化なし
5. **回帰ゼロ**: ベースライン181件が全件維持

#### fr=0.25 / fr=0.20 を不採用とする理由

- fr=0.25 は fr=0.30 から +1件 (q080:p2、正当) のみの追加 (+0.3pp)
- fr=0.20 は fr=0.25 と完全同一 (追加効果ゼロ)
- 閾値を 0.30 より下げてもリターンがほぼなく、min_overlap=3 と direct_recall≥0.15 がボトルネック
- 不必要な閾値緩和は将来のデータ追加時の偽ヒットリスクを高める

#### リスク

- 偽ヒット3件は ovl=3 の最小ラインで発生。synonym_map 拡張時に同パターンの偽ヒットが増える可能性がある
- ただし、θ=0.08 (ovl=2) で発生した26件の爆発とは異なり、ovl=3 が維持されているため構造的安全性は確保

### パラメータ変更

| パラメータ | 現行 (fr=0.35) | 推奨 (fr=0.30) |
|-----------|---------------|---------------|
| `full_recall` 閾値 (primary path) | ≥ 0.35 | ≥ 0.30 |
| `direct_recall` 閾値 | ≥ 0.15 | ≥ 0.15 (変更なし) |
| `_MIN_OVERLAP` | 3 | 3 (変更なし) |
| 演算子回収パス | fr≥0.25, dr≥0.10, ovl≥2 | 変更なし |

変更対象: `detector.py` L975 の `full_recall >= 0.35` → `full_recall >= 0.30`
