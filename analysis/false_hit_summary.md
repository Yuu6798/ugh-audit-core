# 偽ヒット / 偽ミス 不一致分析（HA48）

## 概要

system（cascade baseline）と human アノテーションの命題 hit/miss 不一致を分析。
human は命題ヒット数のみ既知（どの命題が hit かは不明）のため、
TP/FP/FN/TN は上界・下界の範囲として算出。

- **Best case**: system と human の一致を最大化（TP_max = min(sys_hits, human_hits)）
- **Worst case**: 一致を最小化（TP_min = max(0, sys_hits + human_hits - n)）
